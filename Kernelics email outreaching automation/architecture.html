Key Bottleneck: Apollo Basic API Rate Limit

Apollo Basic plan limits you to 5 API requests per minute. This means your OpenClaw sync cycles need to be efficient: batch where possible, cache Apollo data locally, and prioritize reply-detection polls over status syncs. If you scale beyond ~500 active leads, consider upgrading Apollo to Professional for higher rate limits and webhook support (which would let you switch from polling to push-based sync).

OpenClaw Bot — Job Definitions
Each job is a standalone OpenClaw skill/task that runs on a schedule
A
apollo-hubspot-sync
Schedule: Every 15 minutes SoulFlow cron
Steps:
1. GET Apollo sequences → list active contacts per sequence
2. For each contact: get current step, open/reply/bounce status
3. Match to HubSpot contact by email
4. Batch-update HubSpot properties (sequence_step, reply_status, active_tools)
5. If reply detected → trigger Job C (cross-channel coordination)

API calls per cycle: ~2-3 Apollo calls + 1-2 HubSpot batch calls
Handles: Up to 100 contacts per batch (HubSpot limit)
B
lemlist-hubspot-sync
Schedule: Every 15 minutes SoulFlow cron
Steps:
1. GET Lemlist campaign exports → lead statuses
2. Map statuses: emailsSent/emailsOpened/emailsReplied/interested/notInterested
3. Match to HubSpot contact by email
4. Batch-update HubSpot (same properties as Job A)
5. If interested/replied → trigger Job C

API calls per cycle: 1 Lemlist export call + 1-2 HubSpot batch calls
Note: Lemlist has webhooks — can switch to push-based later
C
cross-channel-coordinator
Trigger: Called by Job A or B when reply detected SoulFlow event
Steps:
1. Read HubSpot contact → get active_tools list
2. For each active tool (except the one that got the reply):
  — Apollo: POST remove_contact_ids from sequence
  — Lemlist: DELETE lead from campaign
  — LinkedIn: flag for manual pause (no API)
3. Update HubSpot: active_tools = "", reply_status = "replied"
4. Trigger Job D (task creation)

Critical: This prevents the "already in contact but still getting automated emails" problem
D
hubspot-task-creator
Trigger: Called by Job C after cross-channel cleanup SoulFlow event
Steps:
1. Create HubSpot task:
  — Subject: "Reply from {firstName} {lastName} via {channel}"
  — Body: sequence step #, reply snippet, outreach history
  — Due: today, assigned to you
2. Associate task with contact + deal
3. Create note on contact with full context
4. Move deal to "Engaged" stage

Result: You see the task in HubSpot → handle the reply manually
E
dedup-guard
Trigger: Before any lead is added to a new sequence SoulFlow pre-hook
Steps:
1. Query HubSpot: does this email exist?
2. If exists: check active_tools and reply_status
3. If already in active sequence elsewhere → BLOCK + log
4. If already replied → BLOCK + log
5. If clean → APPROVE, update active_tools in HubSpot

Prevents: Same lead getting emails from Apollo AND Lemlist simultaneously
F
linkedin-manual-sync
Schedule: Daily (or on-demand) SoulFlow cron
Steps:
1. Read Sales Navigator CSV export (you export manually or semi-auto)
2. Parse: connection status, InMail status, DM replies
3. Match to HubSpot contacts by name + company (no email in LinkedIn exports)
4. Update HubSpot: outreach_channel = "both", LinkedIn status in notes
5. If LinkedIn reply detected → trigger Job C

Limitation: No Sales Navigator API — this is the least automated part.
Future upgrade: Lemlist Multichannel Expert can automate LinkedIn steps and feed back to OpenClaw via Lemlist API instead.
